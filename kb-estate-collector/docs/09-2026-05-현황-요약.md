## 8. 운영 / 트러블슈팅 핵심

### 8.1 macOS prefork fork-safety
```
ENV: OBJC_DISABLE_INITIALIZE_FORK_SAFETY=YES
ENV: PYTHONFAULTHANDLER=1
celery worker --pool=prefork --concurrency=4
```
설정 누락 시 SIGSEGV 빈발 → 좀비 task 누적.

### 8.2 RedBeat 동적 cron
- DB(`crawl_jobs.cron_schedule`)에 변경 시 `_sync_redbeat_entry()` 호출 → Redis 즉시 반영
- 정적 task 는 `celery_app.conf.beat_schedule` 에 (예: `cleanup-zombie-runs`)

### 8.3 ON CONFLICT DO NOTHING 적용 지점
- `Transaction` (idx_transaction_unique)
- `ComplexFacility` (uq_facility_complex_type_extid)
- 그 외 KBPrice/Listing 은 SELECT-then-INSERT/UPDATE 로 이미 안전

### 8.4 좀비 cleanup 임계값
- task: 10분 RUNNING/PENDING 초과 → FAILED
- run: 60분 RUNNING 초과 → FAILED
- 5분마다 `cleanup_zombie_runs` 실행

### 8.5 자주 보는 에러 → 원인
| 메시지 | 원인 | 대응 |
|---|---|---|
| `Server disconnected` | KB transient | `kb_base` retry 자동 (3회) |
| `IntegrityError idx_transaction_unique` | UPSERT 미적용 코드 잔재 | tasks.py 의 INSERT 를 ON CONFLICT 로 |
| `kb_area_code not found` | Area 미등록 단지 | `ensure_complex_areas()` 재호출 |
| `integer out of range` (listings.ask_price) | BigInt 마이그 미적용 | `docker compose exec admin-api alembic upgrade head` |
| `PendingRollbackError uq_facility_*` | facility ON CONFLICT 미적용 잔재 | tasks.py:ensure_complex_facilities 확인 |
| run 이 RUNNING 으로 멈춤 | worker 비정상 종료 | 5분 후 자동 cleanup, 또는 수동 호출 |

### 8.6 floor NULL 잔여 이슈
PostgreSQL UNIQUE 인덱스는 NULL ≠ NULL — floor=NULL 거래는 중복 방지 안 됨.
KB 응답에서 floor NULL 은 매우 드물어 실용상 무시.

## 9. 모니터링 SQL

```sql
-- 24h 실패 분포
SELECT error_type, COUNT(*) FROM crawl_tasks
WHERE status='FAILED' AND created_at > now() - interval '24 hours'
GROUP BY error_type ORDER BY 2 DESC;

-- 좀비 후보
SELECT id, run_id, task_key, status, started_at FROM crawl_tasks
WHERE status IN ('RUNNING','PENDING')
  AND started_at < now() - interval '15 minutes';

-- area 0개 단지 (수집 사각지대)
SELECT c.id, c.name FROM complexes c
LEFT JOIN areas a ON a.complex_id = c.id
WHERE c.is_active GROUP BY c.id, c.name HAVING COUNT(a.id) = 0;

-- 단지별 마지막 수집 시각
SELECT c.id, c.name, MAX(t.finished_at) last_collected
FROM complexes c LEFT JOIN crawl_tasks t
  ON t.task_key LIKE 'kb_%_'||c.id||'_%'
WHERE c.is_active GROUP BY c.id, c.name
ORDER BY last_collected DESC NULLS LAST;
```
