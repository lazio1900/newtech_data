# 아키텍처 상세

## 커넥터 상속 구조

```
BaseConnector (base.py)
│  - 재시도 로직 (exponential backoff + jitter)
│  - 레이트 리밋 (분당 요청 수 제한)
│  - 에러 분류 (Network/Auth/Parser/RateLimit/Browser)
│  - collect() = fetch() + parse() 파이프라인
│
├── KBBaseConnector (kb_base.py)
│   │  - 하이브리드 전략 (HTTP → Browser 폴백)
│   │  - httpx 직접 호출 (_fetch_via_http)
│   │  - Playwright 브라우저 폴백 (_fetch_via_browser)
│   │  - KB 공통: 헤더 모방, 만원→원 변환, 날짜 파싱
│   │  - DB에서 kb_complex_id / kb_area_code 조회
│   │
│   ├── KBPriceConnector (kb_price.py)
│   │     fetch(complex_id, area_id) → 시세
│   │     parse() → [{as_of_date, general_price, high_avg_price, low_avg_price}]
│   │
│   ├── KBTransactionConnector (kb_transaction.py)
│   │     fetch(complex_id) → 실거래가 목록
│   │     parse() → [{contract_date, price, exclusive_m2, floor, is_cancelled}]
│   │
│   └── KBListingConnector (kb_listing.py)
│         fetch(complex_id) → 매물 목록
│         parse() → [{source_listing_id, ask_price, exclusive_m2, floor, status}]
│
└── MolitTransactionConnector (molit_transaction.py)
      국토교통부 OpenAPI 기반 (별도 소스)
```

## 데이터 흐름

```
[API 요청 / Celery Beat 스케줄]
    ↓
[Celery Task]
    ↓
[KBxxxConnector.collect()]
    ├── _wait_for_rate_limit()     → 레이트 리밋 준수
    ├── fetch(**kwargs)             → 데이터 가져오기
    │   ├── _fetch_via_http()       → httpx 직접 호출 (1차)
    │   └── _fetch_via_browser()    → Playwright 폴백 (2차)
    └── parse(raw_data)             → 정규화
    ↓
[DB 저장 (SQLAlchemy merge)]
    ├── KBPrice 테이블
    ├── Transaction 테이블
    └── Listing 테이블 (+ stale 매물 REMOVED 처리)
```

## 브라우저 세션 관리

```python
BrowserSessionManager (싱글턴)
├── _initialize()
│   ├── Playwright chromium 실행 (headless)
│   ├── 한국어 로케일, 서울 타임존
│   ├── 랜덤 User-Agent 선택
│   └── stealth 스크립트 주입
├── new_page()  → 새 탭 생성 + 안티 디텍션 적용
└── shutdown()  → 워커 종료 시 브라우저 정리
```

## Celery 태스크 목록

| 태스크 | 설명 | 트리거 |
|--------|------|--------|
| `collect_kb_price_task` | 단지/면적별 시세 수집 | 개별 |
| `collect_kb_transaction_task` | 단지별 실거래가 수집 | 개별 |
| `collect_kb_listing_task` | 단지별 매물 수집 | 개별 |
| `run_kb_price_collection` | 전체 활성 단지 시세 수집 | 일 1회 Beat |
| `run_transaction_collection` | 전체 활성 단지 실거래가 수집 | 일 1회 Beat |
| `run_listing_collection` | 전체 매물수집 활성 단지 | 12시간마다 Beat |
| `discover_complexes_task` | 지역코드로 단지 발견/등록 | API 호출 |
| `run_region_collection` | 지역 기반 전체 수집 (발견+시세+실거래+매물) | API 호출 |

## REST API 엔드포인트

| Method | Path | 설명 |
|--------|------|------|
| POST | `/api/complexes/discover-region` | 지역코드로 단지 자동 발견 |

| POST | `/api/jobs/run-region` | 지역 기반 전체 수집 |
