# 아키텍처 상세

## 커넥터 상속 구조

```
BaseConnector (base.py)
│  - 재시도 로직 (exponential backoff + jitter)
│  - 레이트 리밋 (분당 요청 수 제한)
│  - 에러 분류 (Network/Auth/Parser/RateLimit/Browser)
│  - collect() = fetch() + parse() 파이프라인
│
├── KBBaseConnector (kb_base.py)
│   │  - 하이브리드 전략 (HTTP → Browser 폴백)
│   │  - HTTP/2 httpx 직접 호출 (_fetch_via_http) + transient 재시도(0.5/1/2s)
│   │  - Playwright 브라우저 폴백 (_fetch_via_browser)
│   │  - KB 공통: 헤더 모방, 만원→원 변환, 날짜 파싱
│   │  - DB에서 kb_complex_id / kb_area_code 조회
│   │
│   ├── KBPriceConnector (kb_price.py)
│   │     fetch(complex_id, area_id) → BasePrcInfoNew
│   │     parse() → [{as_of_date, general_price, high_avg_price, low_avg_price}]
│   │
│   ├── KBTransactionConnector (kb_transaction.py)
│   │     fetch(complex_id, area_id) → preSalePrices  ← 5월 변경
│   │     parse() → [{contract_date, price, exclusive_m2, floor}]
│   │
│   ├── KBListingConnector (kb_listing.py)
│   │     fetch(complex_id) → brif (Step 1) + propList/main (Step 2, HTTP/2 필수)
│   │     parse() → [{source_listing_id, ask_price, exclusive_m2, floor, status}]
│   │
│   └── KBFacilityConnector (kb_facility.py)            ← 5월 신규
│         fetch_all(kb_complex_id, lat, lng) →
│         학군(어린이집~고교) + 지하철 + 병원 + OSM 공원
│
├── OSMNatureConnector (osm_nature.py)                  ← 5월 신규
│      Overpass API → 공원/하천 (leisure=park, waterway 등)
│
└── MolitTransactionConnector (molit_transaction.py)
      국토교통부 OpenAPI (잔존, 5월 시점 미사용)
```

> transient 재시도(`RemoteProtocolError`/`ReadTimeout`/`ConnectError` 등)는 `kb_base._fetch_via_http`
> 에 통합되어 모든 KB connector 가 자동 적용. 자세한 내용은
> [08-크롤링-안정성-보강](08-크롤링-안정성-보강-2026-05.md) §3.2 참조.

## 데이터 흐름

```
[API 요청 / Celery Beat 스케줄]
    ↓
[Celery Task]
    ↓
[KBxxxConnector.collect()]
    ├── _wait_for_rate_limit()     → 레이트 리밋 준수
    ├── fetch(**kwargs)             → 데이터 가져오기
    │   ├── _fetch_via_http()       → httpx 직접 호출 (1차)
    │   └── _fetch_via_browser()    → Playwright 폴백 (2차)
    └── parse(raw_data)             → 정규화
    ↓
[DB 저장]
    ├── KBPrice         (SELECT-then-INSERT/UPDATE)
    ├── Transaction     (pg_insert ... ON CONFLICT DO NOTHING)  ← 5월 변경
    ├── Listing         (SELECT-then-INSERT/UPDATE + stale REMOVED 처리)
    └── ComplexFacility (pg_insert ... ON CONFLICT DO NOTHING, 5월 신규)
```

## 브라우저 세션 관리

```python
BrowserSessionManager (싱글턴)
├── _initialize()
│   ├── Playwright chromium 실행 (headless)
│   ├── 한국어 로케일, 서울 타임존
│   ├── 랜덤 User-Agent 선택
│   └── stealth 스크립트 주입
├── new_page()  → 새 탭 생성 + 안티 디텍션 적용
└── shutdown()  → 워커 종료 시 브라우저 정리
```

## Celery 태스크 목록 (5월 현재)

| 태스크 | 설명 | 트리거 |
|--------|------|--------|
| `collect_kb_price_task` | 단지/면적별 시세 + 추출된 최근실거래 | run_kb_collection 서브 |
| `collect_kb_transaction_task` | 단지/면적별 실거래 (preSalePrices) | run_kb_collection 서브 |
| `collect_kb_listing_task` | 단지별 매물 (brif + propList/main) | run_kb_collection 서브 |
| `run_kb_collection` | 단지 리스트 통합 수집 (시세+실거래+매물+facility) | UI 즉시/일괄, scheduled job |
| `discover_complexes_task` | 지역코드로 단지 발견/등록 (5x5 grid) | UI / scoped batch |
| `run_region_collection` | (legacy) 지역 발견 + 수집 | API |
| `run_scheduled_job` | DB cron 진입점 (sido/region/dong 분기) | RedBeat |
| `cleanup_zombie_runs` | 좀비 RUNNING task/run 청소 (10/60분 cutoff) | 5분 주기 정적 beat |

보조 함수 (worker 내부):
- `ensure_complex_detail` — COMPLEX_DETAIL 호출, dong_code/lat/lng 보정
- `ensure_complex_areas` — typInfo 로 면적 자동 등록
- `ensure_complex_facilities` — 학군 cutoff + ON CONFLICT DO NOTHING
