# KB부동산 크롤링 시스템 - 프로젝트 개요

## 목적
KB부동산(kbland.kr)에서 아파트 3가지 데이터를 자동 수집하는 시스템

| 데이터 | 설명 | 커넥터 |
|--------|------|--------|
| **KB 시세** | 일반가, 상위평균가(상한가), 하위평균가(하한가) | `KBPriceConnector` |
| **실거래가** | 계약일, 거래가, 전용면적, 층, 해제여부 | `KBTransactionConnector` |
| **매물정보** | 매물ID, 호가, 전용면적, 층, 상태, 등록일 | `KBListingConnector` |

## 기술 스택
- **언어**: Python 3.11+
- **웹 프레임워크**: FastAPI (관리 API)
- **ORM**: SQLAlchemy 2.0
- **태스크 큐**: Celery + Redis
- **DB**: PostgreSQL
- **브라우저 자동화**: Playwright (chromium)
- **HTTP 클라이언트**: httpx

## 핵심 전략: 하이브리드 크롤링
```
1차 시도: httpx로 직접 API 호출 (빠름, 가벼움)
    ↓ 연속 3회 실패 시
2차 시도: Playwright 브라우저 렌더링 + API 인터셉트 (느리지만 확실)
```

kbland.kr은 SPA(Vue.js)로, 내부 API를 XHR로 호출합니다.
WebFetch 등으로는 HTML 껍데기만 받아올 수 있어서 Playwright가 필수입니다.

## 프로젝트 구조
```
kb-estate-collector/
├── src/
│   ├── browser/                    # [신규] 브라우저 자동화
│   │   ├── stealth.py              # 안티 디텍션 (UA풀, webdriver 위장)
│   │   ├── session_manager.py      # Playwright 싱글턴 세션 관리
│   │   └── api_discovery.py        # KB API 엔드포인트 자동 발견
│   │
│   ├── connectors/                 # 데이터 수집 커넥터
│   │   ├── base.py                 # 추상 베이스 (재시도, 레이트리밋)
│   │   ├── kb_base.py              # [신규] KB 하이브리드 베이스
│   │   ├── kb_endpoints.py         # [신규] API 엔드포인트 정의
│   │   ├── kb_price.py             # [개선] KB 시세 커넥터
│   │   ├── kb_transaction.py       # [신규] KB 실거래가 커넥터
│   │   ├── kb_listing.py           # [개선] KB 매물 커넥터
│   │   └── molit_transaction.py    # 국토교통부 API (기존)
│   │
│   ├── services/                   # [신규] 비즈니스 서비스
│   │   └── complex_discovery.py    # 지역 기반 단지 자동 발견
│   │
│   ├── workers/                    # Celery 태스크
│   │   ├── celery_app.py           # [개선] Beat 스케줄, 브라우저 정리
│   │   └── tasks.py                # [개선] 시세/실거래가/매물/지역수집
│   │
│   ├── models/                     # SQLAlchemy 모델
│   ├── admin_api/                  # FastAPI 라우터
│   └── core/                       # 설정, DB, 로깅
│
├── scripts/
│   └── run_api_discovery.py        # [신규] API 발견 CLI 도구
│
└── docs/                           # [신규] 문서
```
