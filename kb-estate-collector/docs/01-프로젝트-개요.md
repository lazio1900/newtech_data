# KB부동산 크롤링 시스템 - 프로젝트 개요

## 목적
KB부동산(kbland.kr)에서 아파트 3가지 데이터를 자동 수집하는 시스템

| 데이터 | 설명 | 커넥터 |
|--------|------|--------|
| **KB 시세** | 일반가, 상위평균가(상한가), 하위평균가(하한가) | `KBPriceConnector` |
| **실거래가** | 계약일, 거래가, 전용면적, 층, 해제여부 | `KBTransactionConnector` |
| **매물정보** | 매물ID, 호가, 전용면적, 층, 상태, 등록일 | `KBListingConnector` |

## 기술 스택
- **언어**: Python 3.11+
- **웹 프레임워크**: FastAPI (관리 API, port 8000)
- **ORM**: SQLAlchemy 2.0 + Alembic
- **태스크 큐**: Celery + Redis
- **스케줄러**: **RedBeat** (`redbeat.RedBeatScheduler`) — Redis 기반 동적 cron, 변경 즉시 반영
- **DB**: PostgreSQL (port 5433, newtech-platform 과 공유 — ADR-002 ownership separation)
- **브라우저 자동화**: Playwright (chromium) — HTTP 폴백 시
- **HTTP 클라이언트**: httpx (HTTP/2)
- **Timezone**: Asia/Seoul (`enable_utc=False`, `core/time.py:now_kst`)
- **Frontend**: React 19 + Vite (port **5174** — 5173 은 newtech-platform 점유)

> 5월 현황 종합은 [09-2026-05-현황-요약.md](09-2026-05-현황-요약.md) 참조.

## 핵심 전략: 하이브리드 크롤링
```
1차 시도: httpx로 직접 API 호출 (빠름, 가벼움)
    ↓ 연속 3회 실패 시
2차 시도: Playwright 브라우저 렌더링 + API 인터셉트 (느리지만 확실)
```

kbland.kr은 SPA(Vue.js)로, 내부 API를 XHR로 호출합니다.
WebFetch 등으로는 HTML 껍데기만 받아올 수 있어서 Playwright가 필수입니다.
