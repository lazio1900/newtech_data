# KB Estate Collector 사용 가이드

브라우저에서 **http://localhost:5174** 에 접속하면 좌측 사이드바에 4개 메뉴가 있습니다 (2026-05 기준).

```
대시보드      → 전체 현황 요약
단지 관리     → 아파트 단지 등록/발견/관리 + 단지별 시세/실거래/매물/시설 조회
배치 설정     → 시도/시군구/동 단위 cron 자동 수집 (RedBeat 즉시 반영)
실행 이력     → 수집 실행 결과 확인
```

> 데이터 탐색 메뉴는 단지 상세 페이지에 통합되어 사이드바에서 제거됨.
> 시세/실거래/매물/시설(학군·지하철·병원·공원) 모두 단지 상세에서 확인.

---

## 1단계: 단지 등록하기 (단지 관리)

데이터를 수집하려면 먼저 수집 대상 아파트 단지가 DB에 등록되어야 합니다.
등록 방법은 2가지입니다.

### 방법 A: 지역 자동 발견 (추천)

1. 좌측 메뉴에서 **"단지 관리"** 클릭
2. 우측 상단 **"지역 발견"** 버튼 클릭
3. 다이얼로그에서 **시/도 → 시/군/구** 계층형 선택:
   - **시/도 클릭**: 해당 시/도의 모든 시/군/구 전체 선택 (예: 서울 클릭 → 25개 구 전체 선택)
   - **시/도 더블클릭**: 시/군/구 상세 목록 펼침 → 개별 선택 가능
   - 여러 시/도를 동시에 선택 가능 (예: 서울 + 경기 = 전체 선택)
   - 선택 상태 표시: 전체 선택(파란색), 일부 선택(테두리 + 숫자), 미선택
4. 하단 **"발견 시작 (N)"** 버튼 클릭
5. 선택한 지역들을 순차적으로 발견하여 결과를 합산 표시

> **직접 입력**: 상단 입력란에 지역코드(예: `11680`)를 직접 입력하여 발견도 가능

### 전국 지역 코드 (시/도 → 시/군/구)

| 시/도 | 코드 | 주요 시/군/구 |
|-------|------|---------------|
| 서울 | 11 | 강남구(11680), 서초구(11650), 송파구(11710) 등 25개 |
| 경기 | 41 | 분당(41135), 수원 영통(41117), 고양 일산동(41285) 등 27개 |
| 인천 | 28 | 연수구(28185), 남동구(28200), 부평구(28237) 등 8개 |
| 부산 | 26 | 해운대구(26350), 수영구(26500), 동래구(26260) 등 16개 |
| 대구 | 27 | 수성구(27260), 달서구(27290) 등 8개 |
| 대전 | 30 | 유성구(30200), 서구(30170) 등 5개 |
| 광주 | 29 | 광산구(29200), 북구(29170) 등 5개 |
| 울산 | 31 | 남구(31140), 중구(31110) 등 5개 |
| 세종 | 36 | 세종시(36110) |
| 제주 | 50 | 제주시(50110), 서귀포시(50130) |

### 방법 B: 수동 등록

1. **"단지 관리"** → 우측 상단 **"단지 등록"** 버튼 클릭
2. 다이얼로그에서 입력:
   - **단지명**: 예) 래미안퍼스티지
   - **주소**: 예) 서울 서초구 반포동
   - **KB 단지 ID**: KB부동산에서의 단지 고유 번호 (예: 23511)
   - **지역코드**: 행정구역 코드 (예: 11650)
   - **우선순위**: 높음/보통/낮음
3. **"등록"** 클릭

## 2단계: 단지 목록 관리 및 필터링

### 지역 필터

단지 관리 페이지에서 2단계 지역 필터로 원하는 지역의 단지만 볼 수 있습니다:

1. **시/도 버튼** 클릭 (예: 서울) → 해당 시/도 단지만 표시
2. 시/도 선택 후 **시/군/구 버튼** 표시 → 클릭하면 해당 구 단지만 표시
3. 각 버튼에 등록된 단지 수 표시 (예: `강남구 48`)
4. **"전체 보기"** 클릭으로 필터 해제

### 검색

상단 검색창에 **이름, 주소, KB 단지 ID, 지역코드**로 검색 가능:
- 예: "래미안" → 이름에 "래미안"이 포함된 단지만 표시
- 예: "11680" → 강남구 단지만 표시

### 단지 상세

단지 이름을 클릭하면 상세 페이지로 이동합니다.

**상단**: 주소, KB 단지 ID, 우선순위/상태, 지역코드, 매물수집 여부

**하단 탭** (2026-05):

| 탭 | 내용 |
|----|------|
| **면적 타입** | 전용면적(㎡), 공급면적(㎡), 평형, KB 코드 |
| **시세 추이** | 라인 차트(일반가/상위평균/하위평균) + 면적 선택 + CSV |
| **실거래가** | 계약일, 거래가(억원 표시), 전용면적, 층 (preSalePrices 소스) |
| **매물** | 매물ID, 호가, 전용면적, 층, 상태(active/sold/removed) |
| **주변 시설** | 학군(어린이집~고교, 거리 cutoff 적용) / 지하철 / 병원 / 공원 |

## 3단계: 데이터 수집하기

단지가 등록되었으면 이제 실제 크롤링을 실행합니다.

### 방법 A: 단지 선택 후 즉시 수집 (추천)

1. **"단지 관리"** 페이지에서 시/도 → 시/군/구 필터로 대상 지역 선택
2. 단지 목록에서 **체크박스**로 원하는 단지 선택
   - 헤더 체크박스 클릭 → 현재 표시된 단지 **전체 선택**
   - 개별 체크박스로 세부 선택
3. 선택된 단지가 있으면 상단에 **"크롤링 실행 (N)"** 버튼 표시
4. 버튼 클릭 → 선택된 단지들의 시세 + 실거래가 + 매물 일괄 수집 시작
5. 자동으로 **실행 이력 상세 페이지**로 이동하여 진행 상태 확인

### 방법 B: 개별 단지 즉시 수집

단지 목록에서 각 행 우측의 **▶ 버튼** 클릭 → 해당 단지만 즉시 수집

### 방법 C: 배치 설정으로 정기 자동 수집 (2026-05 신규)

1. 좌측 메뉴 **"배치 설정"** 클릭
2. 시도/시군구/동 단위로 cron 표현식 입력
3. 저장 → **RedBeat 가 즉시 동기화** (재기동 불필요)
4. 정해진 시각마다 해당 범위 단지를 자동 수집
5. 같은 화면에서 **"즉시 실행"** 버튼으로 한 번만 트리거도 가능

## 4단계: 수집 결과 확인 (실행 이력)

### 실행 목록

좌측 메뉴에서 **"실행 이력"** 클릭하면 모든 실행이 표시됩니다:

| 컬럼 | 설명 |
|------|------|
| ID | 실행 고유 번호 (클릭하면 상세로 이동) |
| **대상** | 수집 대상 단지명 (예: "래미안강남 외 5개 단지") |
| 상태 | 대기/실행중/성공/부분성공/실패/취소 |
| 시작 시간 | 실행 시작 시각 (**한국시간 KST**) |
| 소요 시간 | 시작~종료까지 걸린 시간 |
| 태스크 | 전체 태스크 수 |
| 성공/실패 | 성공/실패 태스크 수 |

### 실행 상세

실행 ID를 클릭하면 상세 페이지가 표시됩니다.

**상단 요약 카드:**
- 상태, 시작 시간(KST), 완료 시간(KST), 소요 시간

**수집 대상 카드:**
- 크롤링한 단지 목록을 클릭 가능한 태그로 표시
- 각 태그에 단지명 + 지역명 (예: "래미안강남 (강남구)")
- 태그 클릭 시 해당 단지 상세 페이지로 이동

**태스크 목록:**
- 각 단지별 수집 결과를 보여줍니다
- 상태 필터 버튼: 전체 / 성공 / 실패 / 실행중 / 대기

| 컬럼 | 설명 |
|------|------|
| 태스크 키 | 어떤 단지의 어떤 데이터를 수집했는지 |
| 상태 | 성공/실패/실행중/대기/재시도/스킵 |
| 수집 | 수집된 데이터 건수 |
| 저장 | DB에 저장된 건수 |
| 재시도 | 재시도 횟수 |
| 소요시간 | 해당 태스크 소요 시간 |
| 에러 | 실패 시 에러 메시지 |

## 5단계: 수집된 데이터 조회 (단지 상세 내부 탭)

> 2026-05부터 데이터 탐색 메뉴는 사이드바에서 제거되고 **단지 상세 페이지의 탭**으로 이동.

### 단지 상세에서 데이터 보기

1. 좌측 메뉴 **"단지 관리"** → 원하는 단지 클릭 (또는 실행 이력의 대상 태그 클릭)
2. 상단에 단지 기본정보, 하단 5개 탭:

### KB 시세 탭

- **시세 추이 차트**: 일반가(파란선), 상위평균(녹색선), 하위평균(빨간선)의 변화 추이
- **면적 필터**: 좌측 드롭다운에서 면적 타입 선택 (예: 84㎡(25평))
- **데이터 테이블**: 기준일, 일반가, 상위평균, 하위평균, 수집일
- **CSV 내보내기**: 우측 상단 **"CSV 내보내기"** 버튼 → 엑셀에서 열 수 있는 CSV 다운로드

> 가격은 억/만원 단위로 자동 변환됩니다 (예: 50500만원 → "5억 500만")

### 실거래가 탭

- 계약일, 거래가, 전용면적(㎡), 층, 출처
- 최근 200건 표시

### 매물 탭

- 매물ID, 호가, 전용면적, 층, 상태, 수집일
- 상태 필터: active(현재 등록된 매물), sold(매도완료), removed(삭제됨)

## 전체 워크플로우 요약 (2026-05)

```
① 단지 관리 → "지역 발견" → 서울 클릭 (25개 구 전체 선택)
   └→ KB API 5x5 grid 로 서울 전체 아파트 단지 자동 검색 & DB 등록 (dong_code/lat/lng 까지)

② 단지 관리 → 체크박스 선택 → "크롤링 실행" (또는 우측 ▶ 단일 실행)
   └→ run_kb_collection 가 단지마다 시세/실거래/매물/시설(학군·지하철·병원·공원) 수집

③ 실행 이력 → 진행 상태 + 대상 단지 태그 확인 (KST)
   └→ 좀비 task 는 5분마다 자동 청소, transient 오류는 자동 재시도

④ 단지 관리 → 단지 클릭 → 시세/실거래/매물/시설 탭
   └→ 차트/표/CSV 내보내기

⑤ 배치 설정 → 시도/시군구/동 단위 cron 등록
   └→ RedBeat 가 즉시 반영, 정해진 시각마다 자동 수집
```

## 대시보드 활용

**대시보드**(`/` 루트 페이지)에서 전체 현황을 한눈에 확인할 수 있습니다:

| 카드 | 내용 |
|------|------|
| 총 단지 | 등록된 단지 수 (활성 단지 수 포함) |
| 활성 작업 | 현재 활성 상태인 수집 작업 수 |
| 총 실행 | 최근 실행 횟수 |
| 최근 성공 | 마지막으로 성공한 실행의 수집 건수와 시간 |

좌측에 **최근 실행 이력** (최대 5건), 우측에 **수집 현황 바 차트** (성공/실패/스킵 비율)가 표시됩니다.
