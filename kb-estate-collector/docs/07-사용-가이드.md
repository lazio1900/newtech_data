# KB Estate Collector 사용 가이드

브라우저에서 http://localhost:5174 (또는 5173)에 접속하면 좌측 사이드바에 5개 메뉴가 있습니다.

```
대시보드      → 전체 현황 요약
단지 관리     → 아파트 단지 등록/발견/관리
수집 작업     → 크롤링 작업 생성/실행
실행 이력     → 수집 실행 결과 확인
데이터 탐색   → 수집된 시세/실거래가/매물 조회
```

---

## 1단계: 단지 등록하기 (단지 관리)

데이터를 수집하려면 먼저 수집 대상 아파트 단지가 DB에 등록되어야 합니다.
등록 방법은 2가지입니다.

### 방법 A: 지역 자동 발견 (추천)

1. 좌측 메뉴에서 **"단지 관리"** 클릭
2. 우측 상단 **"지역 발견"** 버튼 클릭
3. 다이얼로그가 열리면:
   - 직접 입력: 지역코드 입력란에 `11680` 입력 후 **"발견 시작"** 클릭
   - 또는 아래 서울 구 버튼(강남구, 서초구, 송파구 등 12개) 중 하나 클릭
4. KB API가 해당 지역 좌표 범위 내 아파트 단지를 검색하여 자동 등록

> **서울 구 코드**: 강남구(11680), 서초구(11650), 송파구(11710), 강동구(11740),
> 강서구(11500), 영등포구(11560), 관악구(11620), 은평구(11380),
> 노원구(11350), 동대문구(11230), 용산구(11170), 종로구(11110)

### 방법 B: 수동 등록

1. **"단지 관리"** → 우측 상단 **"단지 등록"** 버튼 클릭
2. 다이얼로그에서 입력:
   - **단지명**: 예) 래미안퍼스티지
   - **주소**: 예) 서울 서초구 반포동
   - **KB 단지 ID**: KB부동산에서의 단지 고유 번호 (예: 23511)
   - **지역코드**: 행정구역 코드 (예: 11650)
   - **우선순위**: 높음/보통/낮음
3. **"등록"** 클릭

---

## 2단계: 단지 상세 확인 (단지 관리 → 상세)

단지 목록에서 **단지 이름을 클릭**하면 상세 페이지로 이동합니다.

### 상세 페이지 구성

상단에 단지 기본 정보가 표시됩니다:
- 주소, KB 단지 ID, 우선순위/상태, 지역코드, 매물수집 여부

하단에 4개 탭이 있습니다:

| 탭 | 내용 |
|----|------|
| **면적 타입** | 전용면적(㎡), 공급면적(㎡), 평형, KB 코드 |
| **시세 추이** | 라인 차트(일반가/상위평균/하위평균) + 데이터 테이블. 면적별 선택 가능 |
| **실거래가** | 계약일, 거래가(억원 표시), 전용면적, 층, 출처 |
| **매물** | 매물ID, 호가, 전용면적, 층, 상태(활성/매도완료/삭제) |

### 단지 수정/삭제

상세 페이지 상단의 **"수정"** / **"삭제"** 버튼으로 가능합니다.
삭제 시 확인 다이얼로그가 표시됩니다.

---

## 3단계: 데이터 수집하기 (수집 작업)

단지가 등록되었으면 이제 실제 크롤링을 실행합니다.

### 방법 A: 지역 전체 수집 (추천)

가장 간단한 방법입니다. 단지 발견 + 시세 + 실거래가 + 매물을 한번에 수집합니다.

1. 좌측 메뉴에서 **"수집 작업"** 클릭
2. 우측 상단 **"지역 수집"** 버튼 클릭
3. 다이얼로그에서:
   - 지역코드 입력 (예: `11680`) 후 **"수집 시작"** 클릭
   - 또는 서울 구 버튼 직접 클릭
4. 토스트 메시지로 "지역 수집이 시작되었습니다" 확인
5. → Celery Worker가 백그라운드에서 수집 작업 수행

### 방법 B: 개별 작업 생성 + 실행

특정 유형의 데이터만 수집하고 싶을 때 사용합니다.

1. **"수집 작업"** → **"작업 생성"** 클릭
2. 다이얼로그에서:
   - **작업명**: 예) 강남구 시세 수집
   - **작업 유형**: 드롭다운에서 선택
     - `KB 시세` — KB부동산 시세 데이터
     - `KB 실거래` — KB부동산 실거래가
     - `KB 매물` — KB부동산 매물 정보
     - `국토부 실거래` — 국토교통부 실거래가 API
   - **활성 여부**: 체크
   - **스케줄(cron)**: 자동 실행 원하면 입력 (예: `0 9 * * *` = 매일 오전 9시)
3. 작업 생성 후 목록에서 해당 작업의 **▶ (재생)** 버튼 클릭 → 즉시 실행

### 작업 관리 버튼

| 버튼 | 기능 | 조건 |
|------|------|------|
| ▶ (재생) | 작업 즉시 실행 | 상태가 "활성"일 때 |
| ⏸ (일시정지) | 작업 일시정지 | 상태가 "활성"일 때 |
| ▶ (재생) | 일시정지된 작업 재개 | 상태가 "일시정지"일 때 |

---

## 4단계: 수집 결과 확인 (실행 이력)

### 실행 목록

1. 좌측 메뉴에서 **"실행 이력"** 클릭
2. 테이블에 모든 실행이 표시됩니다:

| 컬럼 | 설명 |
|------|------|
| ID | 실행 고유 번호 (클릭하면 상세로 이동) |
| 상태 | 대기/실행중/성공/부분성공/실패/취소 |
| 시작 시간 | 실행 시작 시각 |
| 소요 시간 | 시작~종료까지 걸린 시간 |
| 태스크 | 전체 태스크 수 |
| 성공 | 성공한 태스크 수 (녹색) |
| 실패 | 실패한 태스크 수 (빨간색) |

### 실행 상세

실행 ID를 클릭하면 상세 페이지가 표시됩니다.

**상단 요약 카드:**
- 상태, 시작/종료 시간, 소요 시간, 태스크 (총/성공/실패/스킵)

**태스크 목록:**
- 각 단지별 수집 결과를 보여줍니다
- 상태 필터 버튼: 전체 / 성공 / 실패 / 실행중 / 대기

| 컬럼 | 설명 |
|------|------|
| 태스크 키 | 어떤 단지의 어떤 데이터를 수집했는지 |
| 상태 | 성공/실패/실행중/대기/재시도/스킵 |
| 수집 | 수집된 데이터 건수 |
| 저장 | DB에 저장된 건수 |
| 재시도 | 재시도 횟수 |
| 소요시간 | 해당 태스크 소요 시간 |
| 에러 | 실패 시 에러 메시지 |

---

## 5단계: 수집된 데이터 조회 (데이터 탐색)

### 데이터 탐색 페이지

1. 좌측 메뉴에서 **"데이터 탐색"** 클릭
2. 상단 **"단지 선택"** 드롭다운에서 원하는 단지 선택
3. 3개 탭에서 데이터 확인:

### KB 시세 탭

- **시세 추이 차트**: 일반가(파란선), 상위평균(녹색선), 하위평균(빨간선)의 변화 추이
- **면적 필터**: 좌측 드롭다운에서 면적 타입 선택 (예: 84㎡(25평))
- **데이터 테이블**: 기준일, 일반가, 상위평균, 하위평균, 수집일
- **CSV 내보내기**: 우측 상단 **"CSV 내보내기"** 버튼 → 엑셀에서 열 수 있는 CSV 다운로드

> 가격은 억/만원 단위로 자동 변환됩니다 (예: 50500만원 → "5억 500만")

### 실거래가 탭

- 계약일, 거래가, 전용면적(㎡), 층, 출처
- 최근 200건 표시

### 매물 탭

- 매물ID, 호가, 전용면적, 층, 상태, 수집일
- 상태 필터: ACTIVE(현재 등록된 매물), SOLD(매도완료), REMOVED(삭제됨)

## 전체 워크플로우 요약

```
① 단지 관리 → "지역 발견" → 강남구(11680) 선택
   └→ KB API에서 강남구 아파트 단지 자동 검색 & DB 등록

② 수집 작업 → "지역 수집" → 강남구(11680) 선택
   └→ Celery Worker가 백그라운드에서 단지별 시세/실거래가/매물 수집

③ 실행 이력 → 진행 상태 확인
   └→ 성공/실패 건수, 에러 메시지 확인

④ 데이터 탐색 → 단지 선택 → KB시세/실거래가/매물 탭
   └→ 시세 추이 차트 확인, CSV 내보내기
```

또는 더 간단하게:

```
단지 관리에서 "지역 발견"으로 단지 등록
→ 수집 작업에서 "지역 수집"으로 크롤링 시작
→ 실행 이력에서 진행 확인
→ 데이터 탐색에서 결과 확인 + CSV 다운로드
```

## 대시보드 활용

**대시보드**(`/` 루트 페이지)에서 전체 현황을 한눈에 확인할 수 있습니다:

| 카드 | 내용 |
|------|------|
| 총 단지 | 등록된 단지 수 (활성 단지 수 포함) |
| 활성 작업 | 현재 활성 상태인 수집 작업 수 |
| 총 실행 | 최근 실행 횟수 |
| 최근 성공 | 마지막으로 성공한 실행의 수집 건수와 시간 |

좌측에 **최근 실행 이력** (최대 5건), 우측에 **수집 현황 바 차트** (성공/실패/스킵 비율)가 표시됩니다.

## 검색/필터 기능

### 단지 관리
- 상단 검색창에 **이름, 주소, 지역코드**로 검색 가능
- 예: "래미안" 입력 → 이름에 "래미안"이 포함된 단지만 표시
- 예: "11680" 입력 → 강남구 단지만 표시

### 실행 상세
- 태스크 목록에서 상태별 필터 버튼 제공
- **"실패"** 버튼 클릭 → 실패한 태스크만 표시하여 에러 원인 파악

### 데이터 탐색
- KB 시세: 면적별 필터 (드롭다운)
- 매물: 상태별 필터 (ACTIVE/SOLD/REMOVED)
