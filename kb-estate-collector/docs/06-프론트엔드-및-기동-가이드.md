## 3. 화면 구성 (2026-05)

사이드바 4개 메뉴: 대시보드 / 단지 관리 / 배치 설정 / 실행 이력
(데이터 탐색 메뉴는 단지 상세 페이지에 통합되어 사이드바에서 제거)

### 3-1. 대시보드 (`/`)
- 등록 단지 수, 활성 작업 수, 최근 실행 수, 수집 데이터 건수 카드
- 최근 실행 목록 (상태 뱃지, 성공/실패 건수)
- 실행 통계 바 차트

### 3-2. 단지 관리 (`/complexes`)
- 시도 → 시군구 2단계 지역 필터 + 검색
- **"지역 발견"** 버튼 → 5x5 grid 로 시군구 box 커버
- **체크박스 일괄 + 우측 ▶ 단일** — 두 경로 모두 `run_kb_collection` 호출
- 단지 상세 (`/complexes/:id`) — 면적/시세/실거래/매물/시설(학군·지하철·병원·공원) 탭 통합

### 3-3. 배치 설정 (`/batches`) — 5월 신설
- 시도/시군구/동 단위로 cron 설정
- DB 변경 시 RedBeat 즉시 동기화 (`_sync_redbeat_entry`)
- 즉시 실행도 가능 (`POST /scoped/run`)

### 3-4. 실행 이력 (`/runs`)
- 실행 목록 (상태, 시작/종료, 소요시간, 성공/실패 건수, KST)
- 실행 상세 → 태스크 목록 + 대상 단지 태그 (클릭 시 단지 상세로 이동)

### 3-5. 단지 상세 내부 데이터 탐색 (구 `/data` 메뉴 흡수)

탭 5개 (라벨 — value):
- **면적 타입** (`areas`): 전용/공급/평형/KB 코드
- **시세 추이** (`prices`): 라인차트(일반가/상위/하위) + 면적 필터 + CSV 내보내기
- **실거래가** (`transactions`): 계약일, 거래가, 면적, 층
- **매물** (`listings`): 호가, 면적, 층, 상태(ACTIVE/REMOVED) 필터
- **주변 시설** (`facilities`): 학군(거리 cutoff 적용)/지하철/병원/공원

### 호스트(venv) 디버그 모드

PDB/IDE 디버거를 붙이고 싶을 때만 사용. 인프라(postgres/redis) 는 여전히 platform docker 가 떠 있어야 한다. 본 레포 docker compose 와 동시 실행 불가 (8000/5174 포트 충돌).

```bash
# 1) Admin API
source .venv/bin/activate
uvicorn src.admin_api.main:app --host 0.0.0.0 --port 8000 --reload

# 2) Celery worker — macOS 에서는 fork-safety env 필수
OBJC_DISABLE_INITIALIZE_FORK_SAFETY=YES PYTHONFAULTHANDLER=1 \
  celery -A src.workers.celery_app worker --pool=prefork --concurrency=4 --loglevel=info

# 3) Celery beat (RedBeat)
celery -A src.workers.celery_app beat -S redbeat.RedBeatScheduler --loglevel=info

# 4) Frontend
cd frontend && npm install && npm run dev   # http://localhost:5174
```

> Windows 에서 호스트 모드: worker 는 `--pool=solo` 필수 (fork 미지원).

## 7. 사용 시나리오

### 시나리오 1: 강남구 아파트 전체 수집

1. 브라우저에서 http://localhost:5173 접속
2. **단지 관리** 메뉴 클릭
3. **"지역 발견"** 버튼 클릭
4. 서울 강남구 (`11680`) 선택 또는 입력
5. → KB API가 강남구 좌표 범위 내 아파트 단지를 자동 검색하여 DB에 등록
6. **수집 작업** 메뉴 → **"지역 수집"** 버튼 클릭
7. 지역 코드 `11680` 입력, 수집 유형 선택 (시세/실거래가/매물)
8. → Celery가 백그라운드에서 각 단지별 데이터 수집 시작
9. **실행 이력** 메뉴에서 진행 상태 실시간 확인
10. **데이터 탐색** 메뉴에서 단지별 시세 차트/실거래가/매물 확인

### 시나리오 2: 특정 단지 시세 확인

1. **단지 관리** → 단지 목록에서 원하는 단지 클릭
2. 단지 상세 → 면적 정보 확인
3. 시세 탭: 시세 추이 차트 + 데이터 테이블
4. CSV 내보내기 버튼으로 엑셀 분석용 데이터 다운로드

### 시나리오 3: CLI로 직접 수집 실행


## 8. 트러블슈팅

### PostgreSQL 연결 실패 (psycopg2 UnicodeDecodeError)

```
UnicodeDecodeError: 'utf-8' codec can't decode byte 0xb8 in position 63
```

**원인**: 로컬에 PostgreSQL이 설치되어 있어 포트 5432 충돌. Python이 Docker 컨테이너가 아닌 로컬 PostgreSQL에 연결되었고, 한국어 Windows(cp949)에서 libpq 에러 메시지 인코딩 불일치 발생.

**해결**:
1. 본 레포 docker-compose 에는 postgres/redis 가 더 이상 없다. platform 측 compose 가 5433/6379 로 노출 중인지 먼저 확인 (`docker ps | grep newtech-platform-postgres`).
2. 호스트 venv 모드라면 `.env` 의 `DATABASE_URL` 이 `postgresql://kb_user:change-me-in-production@localhost:5433/kb_estate` 인지 확인 (비밀번호는 platform `.env` 의 `POSTGRES_PASSWORD` 와 일치).
3. 본 레포 컨테이너가 platform 네트워크에 합류 못 했다면 `docker compose down && docker compose up -d` 로 재기동.

### Alembic versions 디렉토리 없음

```
FileNotFoundError: src/alembic/versions/...
```

**해결**: `mkdir src/alembic/versions` 후 재실행

### Alembic script_location 오류

```
FAILED: No such file or directory: '%(here)s/env.py'
```

**해결**: `alembic.ini`의 `script_location`을 `src/alembic`으로 수정

### Vite 포트 충돌

```
Port 5173 is in use, trying another one...
```

**해결**: 자동으로 5174 등 다음 포트 사용. 정상 동작에 영향 없음.

### Celery Windows 오류

```
ValueError: not enough values to unpack (expected 3, got 0)
```

**해결**: `--pool=solo` 옵션 추가. Windows에서는 fork가 지원되지 않아 solo pool 필수.

## 9. 환경 설정 파일

### .env (호스트 venv 디버그 모드 전용)

도커 모드에서는 `docker-compose.yml` 의 `environment` 가 우선하므로 `.env` 는 참조되지 않는다.

```env
# Database (platform 의 postgres 5433 에 호스트로 붙음)
DATABASE_URL=postgresql://kb_user:change-me-in-production@localhost:5433/kb_estate

# Redis
REDIS_URL=redis://localhost:6379/0

# Celery
CELERY_BROKER_URL=redis://localhost:6379/0
CELERY_RESULT_BACKEND=redis://localhost:6379/0

# API
API_HOST=0.0.0.0
API_PORT=8000

# Logging
LOG_LEVEL=INFO
```

비밀번호 `change-me-in-production` 은 `newtech-platform/.env` 의 `POSTGRES_PASSWORD` 와 동일해야 한다.

### docker-compose.yml 핵심 설정

본 레포 compose 는 admin-api / celery-worker / celery-beat / frontend 네 서비스만 정의하고, postgres/redis 는 platform 네트워크를 외부 참조로 합류해 공유한다.

```yaml
services:
  admin-api:
    build: { context: ., dockerfile: docker/api.Dockerfile }
    ports: ["8000:8000"]
    environment:
      DATABASE_URL: postgresql://kb_user:change-me-in-production@postgres:5432/kb_estate
      REDIS_URL: redis://redis:6379/0
      ...
  # celery-worker / celery-beat / frontend ...

networks:
  default:
    name: newtech-platform_default
    external: true
```
